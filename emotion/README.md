Emotion detection module for the COS30082 facial recognition project. Takes a cropped face image and returns one of seven emotion labels (angry, disgust, fear, happy, neutral, sad, surprise) with a confidence score. Built on MobileNetV2 with ImageNet-pretrained weights, fine-tuned on FER-2013 using TensorFlow and Keras.

The folder is organised as importable Python modules under src and one driver notebook under notebooks. The notebook is the runtime in Colab and produces the plots used in the report. The modules contain all reusable logic so the UI code can import the predictor directly without copy-pasting cells.

Training follows a two-phase transfer-learning recipe. The first phase freezes the entire backbone and trains only the new seven-class classification head, giving the head sensible weights before the backbone is touched. The second phase unfreezes the top portion of the backbone and continues training at a much smaller learning rate, letting the high-level features adapt to facial expressions without destroying the low-level features learned from ImageNet. Batch-normalisation layers in the base model are kept in inference mode throughout, so their pretrained running statistics are not corrupted during fine-tuning.

Class weighting is deliberately not used. The natural FER-2013 imbalance mirrors the real-world emotion distribution in workplace and attendance settings, where happy and neutral faces dominate and disgust is rare. Weighting would inflate minority-class recall at the cost of precision, which is the wrong trade-off when false-positive emotion readings carry more cost than missed ones.

The dataset lives on Google Drive and is downloaded from Kaggle on the first run. Trained model weights and report figures are also written to Drive so they survive Colab disconnects. Source code lives on GitHub. The Colab session clones the repo each time, mounts Drive, and imports the modules from the cloned working tree. Nothing important is stored on the Colab VM itself.

To run, mount Drive and upload a Kaggle API token to the Drive root. Open the notebook in Colab with a GPU runtime and execute cells in order. The first run downloads the dataset; subsequent runs skip the download. Training runs in two phases with checkpoints saved every epoch. The best checkpoint by validation accuracy is also saved separately and is the one used at inference time.

The UI integration point is a single class exposed by the predict module. It loads a checkpoint once at startup and exposes a method that takes a face crop and returns a label, confidence, and the full probability distribution across all seven classes. It handles both OpenCV-style BGR images and PIL or MediaPipe RGB images via a constructor flag, which avoids the common colour-space-swap bug at the integration boundary.

Expected test accuracy on FER-2013 with this recipe is in the low sixties. State-of-the-art on this dataset is around seventy-five percent and human accuracy is roughly sixty-five percent, so the ceiling is set more by the noisy labels and low image resolution than by the model. The confusion matrix typically shows the well-documented confusion pairs of fear with surprise and sad with neutral, which the literature attributes to genuine visual ambiguity in the dataset rather than model deficiency.
